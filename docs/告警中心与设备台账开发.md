# 告警中心与设备台账开发报告

## 1. 完成内容

本次在现有智慧路灯控制台基础上完善告警中心和设备台账能力，使系统从“能查看数据”提升到“能按运维流程筛选、定位、处理和复盘”。

- 告警中心支持按告警级别、处理状态、设备关键字和日期范围筛选。
- 告警详情展示设备位置、发生时间、处理状态、处理人、处理时间和处理备注。
- 告警处理需要填写备注，处理后刷新总览、告警列表和审计日志。
- 告警统计展示未处理、中优先级、已处理和高优先级数量。
- 设备台账支持设备 ID/名称/位置搜索、在线状态筛选、新增设备和详情跳转。
- 设备列表展示在线状态、灯具状态、当前功率、今日能耗、最后心跳和故障预判风险。
- 设备详情展示基础信息、光照趋势、控制记录、关联告警、阈值入口和故障预判结果。

## 2. 页面能力

| 页面 | 能力 |
| --- | --- |
| 设备管理 | 搜索设备、状态筛选、设备统计、新增设备、查看详情 |
| 设备详情 | 基础档案、实时光照、灯具状态、控制面板、控制记录、关联告警、故障预判 |
| 告警日志 | 级别筛选、状态筛选、设备筛选、日期筛选、告警统计、详情查看 |
| 告警详情 | 处理元数据展示、处理备注输入、推荐操作、跳转设备详情 |

## 3. 后端接口

本次继续复用现有 REST API，不破坏原有前端调用：

| 接口 | 用途 |
| --- | --- |
| `GET /api/devices` | 获取设备台账列表 |
| `GET /api/devices/{id}` | 获取设备详情、阈值、告警和控制记录 |
| `POST /api/devices` | 新增设备，仅 `admin` 可用 |
| `GET /api/alarms` | 获取告警列表 |
| `PUT /api/alarms/{id}/handle` | 提交处理备注并标记告警已处理 |
| `GET /api/overview` | 获取总览、设备、告警、阈值和故障预判 |

## 4. 数据字段

| 数据 | 关键字段 |
| --- | --- |
| 设备 | `id`、`name`、`location`、`onlineStatus`、`lampStatus`、`autoMode`、`lastHeartbeatAt` |
| 告警 | `alarmType`、`alarmLevel`、`alarmContent`、`handled`、`handledBy`、`handledAt`、`handleRemark` |
| 控制记录 | `command`、`source`、`result`、`createdAt` |
| 故障预判 | `riskType`、`riskLevel`、`reason`、`suggestedAction`、`evidence` |

## 5. 权限控制

| 角色 | 权限 |
| --- | --- |
| `admin` | 查看设备、查看告警、新增设备、处理告警、查看审计 |
| `operator` | 查看设备、查看告警、处理告警、控灯和调整阈值 |
| `viewer` | 只读查看设备、告警和总览 |
