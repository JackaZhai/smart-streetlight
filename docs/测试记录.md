## 人工联调流程

1. `cp .env.example .env`
2. `docker compose -f deploy/docker-compose.yml up -d`
3. `npm run dev:backend`
4. `npm run dev:frontend`
5. `npm run dev:mock`
6. 打开 `http://localhost:5173`
7. 检查实时光照、历史趋势、设备表格、告警列表和智能问答。
8. 点击“开灯/关灯”，确认 mock-device 输出执行日志。

## MySQL 验收点

1. `docker compose -f deploy/docker-compose.yml ps` 显示 `smart-streetlight-frontend`、`smart-streetlight-backend`、`smart-streetlight-mysql` 和 `smart-streetlight-emqx` 均运行。
2. `curl -sS http://127.0.0.1:4000/health` 返回 `storageDriver=mysql`。
3. `POST /api/auth/login` 使用 `admin/admin123` 能返回 token。
4. 未携带 token 访问 `/api/overview` 返回 `401`。
5. 携带 token 启动 `mock-device` 后，`/api/overview` 能看到最新光照和在线设备。
6. 重启后端后，设备、阈值、告警和光照历史仍可从 MySQL 读取。

## 全栈 Docker Compose 验收点

1. `docker compose -f deploy/docker-compose.yml up -d --build` 能构建并启动四个服务。
2. `curl -sS -I http://127.0.0.1:18080` 返回 `200 OK`。
3. `curl -sS http://127.0.0.1:18080/health` 通过前端 Nginx 反代到后端，并返回 `storageDriver=mysql`。
4. 浏览器访问 `http://127.0.0.1:18080` 能打开前端页面。
5. 登录页使用 `admin/admin123` 可进入控制台。
6. 配置 `AUTH_USERS_JSON` 后，`operator` 可执行运维写操作，`viewer` 写操作返回 `403`。
7. 管理员访问 `/api/audit-logs` 可看到成功写操作和被拒绝操作的审计记录。
8. `npm run db:backup -- --dry-run` 能输出 `mysqldump --single-transaction` 备份命令。
9. `npm run db:restore -- --input backups/mysql/example.sql --dry-run` 能输出恢复命令；未传入 `--yes` 的真实恢复会被拒绝。
