# 测试记录

## 计划验证项

| 项目 | 命令 | 目标 |
| --- | --- | --- |
| 后端领域测试 | `npm --workspace backend test` | 验证遥测入库、自动控制、离线告警和阈值统计 |
| MySQL 集成测试 | `TEST_DATABASE_URL=mysql://streetlight:streetlight_pass@127.0.0.1:3307/smart_streetlight npm --workspace backend test -- tests/mysqlStore.test.ts` | 验证 MySQL 建表、seed、更新和持久化 |
| 全量测试 | `npm test` | 运行所有 workspace 测试 |
| 类型检查 | `npm run typecheck` | 验证 TypeScript 类型 |
| 构建 | `npm run build` | 验证前端、后端、mock-device 可编译 |

## 人工联调流程

1. `cp .env.example .env`
2. `docker compose -f deploy/docker-compose.yml up -d`
3. `npm run dev:backend`
4. `npm run dev:frontend`
5. `npm run dev:mock`
6. 打开 `http://localhost:5173`
7. 检查实时光照、历史趋势、设备表格、告警列表和智能问答。
8. 点击“开灯/关灯”，确认 mock-device 输出执行日志。

## MySQL 验收点

1. `docker compose -f deploy/docker-compose.yml ps` 显示 `smart-streetlight-mysql` 和 `smart-streetlight-emqx` 均运行。
2. `curl -sS http://127.0.0.1:4000/health` 返回 `storageDriver=mysql`。
3. 启动 `mock-device` 后，`/api/overview` 能看到最新光照和在线设备。
4. 重启后端后，设备、阈值、告警和光照历史仍可从 MySQL 读取。
