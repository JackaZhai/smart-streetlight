# 自动控灯与手动控灯逻辑说明

## 1. 功能目标

自动控灯和手动控灯用于完成智慧路灯系统的核心控制闭环：

- 自动控灯：系统根据实时光照和阈值配置自动判断是否开灯或关灯。
- 手动控灯：管理员或运维员在前端点击开灯、关灯按钮后，后端下发 MQTT 控制指令到设备。

该功能连接了光照采集、阈值规则、设备状态、控制日志、审计日志、MQTT 指令下发和前端控制面板。

## 2. 自动控灯逻辑

自动控灯由后端领域函数 `applyTelemetry()` 和 `evaluateAutomation()` 完成。

处理流程：

```text
设备 MQTT 上报光照
  -> MqttBridge 接收 telemetry
  -> applyTelemetry() 更新设备状态和光照历史
  -> evaluateAutomation() 判断阈值
  -> 生成 TURN_ON / TURN_OFF 自动指令
  -> MqttBridge 发布 streetlight/{deviceId}/command
  -> 写入 controlLogs
  -> Socket.IO 推送前端刷新
```

自动控制判断规则：

| 条件 | 动作 |
| --- | --- |
| 光照值 `< lowThreshold` 且当前灯为 `OFF` | 自动下发 `TURN_ON` |
| 光照值 `> highThreshold` 且当前灯为 `ON` | 自动下发 `TURN_OFF` |
| 阈值规则未启用 | 不自动控制 |
| 设备离线 | 不自动控制 |
| 设备关闭自动模式 | 不自动控制 |

自动控制生成的控制日志字段：

| 字段 | 说明 |
| --- | --- |
| `deviceId` | 被控制设备编号 |
| `command` | `TURN_ON` 或 `TURN_OFF` |
| `source` | 固定为 `auto` |
| `result` | 初始为 `QUEUED` |
| `createdAt` | 控制指令生成时间 |

## 3. 手动控灯逻辑

手动控灯入口在前端设备控制面板，前端调用：

```text
frontend/src/components/DeviceControl.vue
  -> sendCommand(deviceId, command)
  -> POST /api/devices/{id}/commands
```

后端接口位于 `backend/src/routes/api.ts`：

1. 校验登录 token。
2. 校验角色权限，只有 `admin` 和 `operator` 可执行。
3. 校验指令只能是 `TURN_ON` 或 `TURN_OFF`。
4. 调用 `applyManualCommand()` 更新设备灯光状态。
5. 写入 `controlLogs` 控制日志。
6. 写入 `auditLogs` 审计日志，记录 `command.dispatch`。
7. 调用 `mqttBridge.publishCommand()` 向设备发布 MQTT 指令。
8. 通过 Socket.IO 推送前端刷新。

手动控制权限：

| 角色 | 是否可控灯 |
| --- | --- |
| `admin` | 可以 |
| `operator` | 可以 |
| `viewer` | 不可以 |

手动控制请求示例：

```json
{
  "command": "TURN_ON"
}
```

响应示例：

```json
{
  "message": "控制指令已下发",
  "command": "TURN_ON",
  "device": {
    "id": "SL-001",
    "lampStatus": "ON"
  }
}
```

## 4. MQTT 指令下发

自动控制和手动控制最终都会通过 MQTT 下发到设备：

| Topic | 方向 | 说明 |
| --- | --- | --- |
| `streetlight/{deviceId}/command` | 后端 -> 设备 | 下发 `TURN_ON` 或 `TURN_OFF` |
| `streetlight/{deviceId}/command/reply` | 设备 -> 后端 | 设备回传执行结果 |

自动控制下发时 `source=auto`，手动控制下发时 `source=manual`。

## 5. 前端交互

前端控制相关位置：

| 文件 | 说明 |
| --- | --- |
| `frontend/src/components/DeviceControl.vue` | 设备详情控制面板，包含开灯、关灯按钮和阈值设置 |
| `frontend/src/services/api.ts` | `sendCommand()` 调用后端控制接口 |
| `frontend/src/App.vue` | 展示控制记录、权限状态、设备详情和阈值规则 |

前端会根据当前登录角色控制按钮可用性：

- `admin`、`operator` 可点击开灯、关灯。
- `viewer` 只能查看，按钮禁用。

## 6. 验收方式

本地测试：

```bash
npm --workspace backend test -- streetlight.test.ts
npm --workspace backend test -- authorization.test.ts validation.test.ts
```

测试覆盖点：

- 光照低于开灯阈值时自动生成 `TURN_ON`。
- 光照高于关灯阈值时自动生成 `TURN_OFF`。
- 手动开灯、关灯会更新设备灯光状态并写入控制日志。
- 非法控制指令会被参数校验拒绝。
- `viewer` 执行控灯会返回 `403`，并写入拒绝审计日志。

人工验收路径：

1. 使用 `admin / admin123` 登录系统。
2. 进入设备详情页。
3. 点击“开灯”或“关灯”。
4. 确认设备灯光状态变化。
5. 确认控制记录新增。
6. 启动 `mock-device` 后，确认模拟设备收到 MQTT 指令并输出执行日志。
7. 使用 `viewer` 登录后确认控灯按钮不可用。
