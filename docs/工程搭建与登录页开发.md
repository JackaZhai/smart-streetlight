# 工程搭建与登录页开发说明

## 1. 工程搭建完成内容

当前项目已经搭建为 TypeScript 全栈工程，根目录通过 npm workspace 管理三个子模块：

| 模块 | 路径 | 说明 |
| --- | --- | --- |
| 前端 | `frontend/` | Vue 3、Vite、TypeScript、ECharts，实现系统页面和实时数据展示 |
| 后端 | `backend/` | Node.js、Express、TypeScript、Socket.IO，实现 API、鉴权、MQTT 接入和业务逻辑 |
| 模拟设备 | `mock-device/` | Node.js、mqtt.js，用于模拟路灯设备上报数据和接收控制指令 |

根目录已经提供统一脚本：

| 命令 | 作用 |
| --- | --- |
| `npm run dev:frontend` | 启动前端开发服务 |
| `npm run dev:backend` | 启动后端 API 服务 |
| `npm run dev:mock` | 启动模拟设备 |
| `npm test` | 运行运维脚本测试和各 workspace 测试 |
| `npm run typecheck` | 运行 TypeScript 类型检查 |
| `npm run build` | 构建前端、后端和模拟设备 |

项目环境变量集中在 `.env.example`，已经覆盖端口、MQTT、MySQL、管理员账号、JWT、扩展用户等配置。开发时复制 `.env.example` 为 `.env` 后即可按 README 启动本地工程。

## 2. 登录页开发完成内容

登录页位于 `frontend/src/components/LoginPanel.vue`，当前已经完成以下能力：

- 提供账号、密码输入框，并默认填入管理员账号 `admin`。
- 输入账号和密码前，登录按钮保持禁用，避免提交空表单。
- 登录过程中显示“登录中”状态，并禁用重复提交。
- 登录失败时展示错误提示，错误区域带有 `role="alert"` 和 `aria-live="polite"`，便于辅助技术识别。
- 页面展示演示账号信息：`admin / admin123`、`operator / operator123`、`viewer / viewer123`。
- 登录成功后通过前端 `login()` 调用后端 `/api/auth/login`，保存 token 和用户角色信息。
- 登录成功后 Socket.IO 会携带 token 建立实时连接，用于后续接收 `state:update` 数据。

## 3. 登录链路对接

前端登录调用链路：

```text
LoginPanel.vue -> frontend/src/services/api.ts login() -> POST /api/auth/login
```

后端登录接口位于 `backend/src/routes/auth.ts`，成功后返回：

```json
{
  "token": "<signed-token>",
  "user": {
    "username": "admin",
    "role": "admin"
  },
  "expiresAt": "2026-07-03T01:00:00.000Z"
}
```

登录后的 token 会自动加入后续 API 请求头：

```http
Authorization: Bearer <token>
```

## 4. 验收方式

工程搭建和登录页开发可通过以下命令验收：

```bash
npm --workspace frontend test -- LoginPanel.test.ts
npm --workspace frontend test
npm run typecheck
npm run build
```

浏览器验收路径：

1. 启动后端和前端。
2. 打开 `http://localhost:5173`。
3. 确认登录页展示系统名称、账号、密码、演示账号说明。
4. 密码为空时登录按钮不可点击。
5. 输入 `admin / admin123` 后可以进入智慧路灯控制台。
6. 输入错误密码时页面显示登录失败提示。
