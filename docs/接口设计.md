# 接口设计

## REST API

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 后端健康检查 |
| POST | `/api/auth/login` | 登录并获取访问令牌 |
| GET | `/api/auth/me` | 查询当前登录用户 |
| GET | `/api/overview` | 获取总览、设备、最新光照、告警和阈值 |
| GET | `/api/devices` | 获取设备列表 |
| GET | `/api/audit-logs` | 查询最近审计日志，要求 `admin` |
| GET | `/api/devices/{id}` | 获取设备详情 |
| POST | `/api/devices` | 新增设备，要求 `admin` |
| GET | `/api/devices/{id}/latest-light` | 获取最新光照 |
| GET | `/api/devices/{id}/light-history` | 获取历史光照 |
| GET | `/api/devices/{id}/threshold` | 获取阈值 |
| PUT | `/api/devices/{id}/threshold` | 更新阈值，要求 `admin` 或 `operator` |
| POST | `/api/devices/{id}/commands` | 下发开灯/关灯指令，要求 `admin` 或 `operator` |
| GET | `/api/alarms` | 查询告警 |
| PUT | `/api/alarms/{id}/handle` | 标记告警已处理，要求 `admin` 或 `operator` |
| POST | `/api/agent/chat` | 智能问答 |

除 `/health` 和 `/api/auth/login` 外，所有 `/api` 接口都需要请求头：

```http
Authorization: Bearer <token>
```

角色说明：

| 角色 | 权限 |
| --- | --- |
| `admin` | 所有读写接口、设备新增、审计日志查询 |
| `operator` | 普通查询、阈值调整、开关灯控制、告警处理 |
| `viewer` | 普通查询和智能问答，只读 |

## 健康检查响应

```json
{
  "ok": true,
  "service": "smart-streetlight-backend",
  "storageDriver": "mysql",
  "timestamp": "2026-07-02T01:00:00.000Z"
}
```

## 登录请求

```json
{
  "username": "admin",
  "password": "admin123"
}
```

登录成功响应：

```json
{
  "token": "<signed-token>",
  "user": {
    "username": "admin",
    "role": "admin"
  },
  "expiresAt": "2026-07-03T01:00:00.000Z"
}
```

## 审计日志响应

```json
[
  {
    "id": "audit-xxxx",
    "actorUsername": "operator",
    "actorRole": "operator",
    "action": "command.dispatch",
    "targetType": "device",
    "targetId": "SL-001",
    "result": "SUCCESS",
    "detail": "TURN_ON",
    "createdAt": "2026-07-02T01:00:00.000Z"
  }
]
```

审计日志记录成功的设备新增、阈值调整、手动控制、告警处理，以及被角色权限拒绝的写操作。

## 控制指令请求

```json
{
  "command": "TURN_ON"
}
```

`command` 可取值：

- `TURN_ON`
- `TURN_OFF`

## 智能问答请求

```json
{
  "question": "设备离线后应该怎么排查？"
}
```

## Socket.IO 事件

| 事件 | 方向 | 说明 |
| --- | --- | --- |
| `state:update` | 后端到前端 | 推送总览数据、设备状态、告警和阈值 |

Socket.IO 连接需要在 `auth.token` 中携带登录 token。
