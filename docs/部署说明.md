# 部署说明

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| Docker | 24 或以上 |
| Docker Compose | v2 或以上 |
| 端口 | `18080`、`4000`、`1883`、`18083`、`3307` 未被占用 |
| 磁盘 | 至少 2 GB 可用空间，用于镜像和 MySQL 数据卷 |

## 2. 一键启动

在项目根目录执行：

```bash
docker compose -f deploy/docker-compose.yml up -d --build
```

默认前端端口是 `18080`。如需改为其他端口，可以在命令前设置 `FRONTEND_PORT`：

```bash
FRONTEND_PORT=8080 docker compose -f deploy/docker-compose.yml up -d --build
```

Compose 会启动四个服务：

| 服务 | 容器名 | 说明 |
| --- | --- | --- |
| `frontend` | `smart-streetlight-frontend` | Nginx 静态前端，反代 API 和 Socket.IO |
| `backend` | `smart-streetlight-backend` | Express API、MQTT bridge、自动控制规则 |
| `emqx` | `smart-streetlight-emqx` | MQTT broker |
| `mysql` | `smart-streetlight-mysql` | MySQL 8 持久化数据库 |

## 3. 访问地址

| 项目 | 地址 |
| --- | --- |
| 前端系统 | `http://localhost:18080` |
| 后端健康检查 | `http://localhost:4000/health` |
| EMQX 控制台 | `http://localhost:18083` |
| MySQL | `127.0.0.1:3307` |

EMQX 默认账号密码：

```text
admin / public
```

MySQL 默认连接信息：

```text
database: smart_streetlight
username: streetlight
password: streetlight_pass
port: 3307
```

## 4. 验收命令

检查容器状态：

```bash
docker compose -f deploy/docker-compose.yml ps
```

检查后端健康状态：

```bash
curl -sS http://127.0.0.1:4000/health
```

期望包含：

```json
{
  "ok": true,
  "storageDriver": "mysql"
}
```

检查前端入口：

```bash
curl -sS -I http://127.0.0.1:18080
```

期望返回 `HTTP/1.1 200 OK`。

## 5. 模拟设备联调

如需验证 MQTT 数据链路，在宿主机执行：

```bash
npm install
MQTT_URL=mqtt://127.0.0.1:1883 npm run dev:mock
```

随后访问：

```bash
curl -sS http://127.0.0.1:4000/api/overview
```

应能看到设备在线状态和最新光照数据。

## 6. 常用运维命令

查看日志：

```bash
docker compose -f deploy/docker-compose.yml logs -f backend
docker compose -f deploy/docker-compose.yml logs -f frontend
```

重启服务：

```bash
docker compose -f deploy/docker-compose.yml restart backend frontend
```

停止服务但保留 MySQL 数据：

```bash
docker compose -f deploy/docker-compose.yml down
```

停止服务并清空 MySQL 数据卷：

```bash
docker compose -f deploy/docker-compose.yml down -v
```

## 7. 生产注意事项

- 当前默认密码用于课程/实习交付，正式上线前必须更换 MySQL 和 EMQX 密码。
- 当前接口尚未加入登录鉴权，生产公网部署前应先完成认证和权限控制。
- MySQL 数据保存在 Docker volume `deploy_mysql-data`，迁移服务器前需要备份该 volume。
