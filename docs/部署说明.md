# 部署说明

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| Docker | 24 或以上 |
| Docker Compose | v2 或以上 |
| 端口 | `18080`、`4000`、`1883`、`18083`、`3307` 未被占用 |
| 磁盘 | 至少 2 GB 可用空间，用于镜像和 MySQL 数据卷 |

## 2. 从 0 启动总流程

如果是在一台新电脑或新服务器上第一次启动，按下面顺序执行，不要只启动前端。前端页面上的登录、控制、告警处理、区域批量控制、趋势数据和 AI 问答都依赖后端 API；后端又依赖 MQTT 和数据库。

完整链路如下：

```text
浏览器前端 -> 后端 API / Socket.IO -> MySQL
                         -> EMQX MQTT -> mock-device 模拟设备
                         -> 本地 RAG / MaxKB 问答
```

从 0 启动前先确认：

```bash
node -v
npm -v
docker --version
docker compose version
```

要求 Node.js `20.19.0` 或以上，Docker `24` 或以上，Docker Compose v2 或以上。

获取代码后进入项目根目录：

```bash
git clone <仓库地址> smart-streetlight
cd smart-streetlight
```

如果已经在项目目录中，直接从 `cd smart-streetlight` 后的步骤开始。

首次启动必须准备环境变量：

```bash
cp .env.example .env
```

课程演示可直接使用默认账号。正式交付前至少修改 `.env` 中的 `ADMIN_PASSWORD`、`JWT_SECRET`，并按需要配置 `AUTH_USERS_JSON`、`MAXKB_API_KEY`、`MAXKB_BASE_URL`。

后续有两种启动方式：

| 方式 | 适用场景 | 启动结果 |
| --- | --- | --- |
| Docker 一键启动 | 演示、交付、服务器部署 | 前端、后端、EMQX、MySQL 全部由 Compose 托管 |
| 本地开发启动 | 改前端/后端代码、调试功能 | EMQX 和 MySQL 用 Docker，前端/后端/mock-device 在本机热更新 |

## 3. Docker 一键启动

在项目根目录执行：

```bash
docker compose -f deploy/docker-compose.yml up -d --build
```

默认前端端口是 `18080`。如需改为其他端口，可以在命令前设置 `FRONTEND_PORT`：

```bash
FRONTEND_PORT=8080 docker compose -f deploy/docker-compose.yml up -d --build
```

Compose 会启动四个服务：

| 服务 | 容器名 | 说明 |
| --- | --- | --- |
| `frontend` | `smart-streetlight-frontend` | Nginx 静态前端，反代 API 和 Socket.IO |
| `backend` | `smart-streetlight-backend` | Express API、MQTT bridge、自动控制规则 |
| `emqx` | `smart-streetlight-emqx` | MQTT broker |
| `mysql` | `smart-streetlight-mysql` | MySQL 8 持久化数据库 |

等待 30 到 60 秒后检查容器状态：

```bash
docker compose -f deploy/docker-compose.yml ps
```

如果 `backend` 一直不是 healthy，优先查看后端日志：

```bash
docker compose -f deploy/docker-compose.yml logs -f backend
```

## 4. 本地开发从 0 启动

本地开发模式适合修改代码。推荐开 4 个终端窗口，顺序不能反。

第 1 步，安装 workspace 依赖：

```bash
npm install
```

第 2 步，只启动基础设施：

```bash
docker compose -f deploy/docker-compose.yml up -d emqx mysql
```

第 3 步，启动后端 API：

```bash
npm run dev:backend
```

后端默认读取 `.env`，监听 `4000`，连接：

```text
MQTT_URL=mqtt://127.0.0.1:1883
DATABASE_URL=mysql://streetlight:streetlight_pass@127.0.0.1:3307/smart_streetlight
```

第 4 步，启动前端：

```bash
npm run dev:frontend
```

前端默认地址是 `http://localhost:5173`。开发模式下 `/api` 和 `/health` 会通过 Vite 代理到 `http://localhost:4000`，Socket.IO 也会连接后端 `4000`。

第 5 步，启动模拟设备：

```bash
npm run dev:mock
```

mock-device 会连接 EMQX，持续上报光照和心跳，并接收后端下发的开灯/关灯指令。只有启动 mock-device 后，实时光照、在线状态、自动控灯和告警变化才会持续刷新。

本地开发访问地址：

| 项目 | 地址 |
| --- | --- |
| 前端开发页 | `http://localhost:5173` |
| 后端健康检查 | `http://localhost:4000/health` |
| EMQX 控制台 | `http://localhost:18083` |
| MySQL | `127.0.0.1:3307` |

## 5. 访问地址

| 项目 | 地址 |
| --- | --- |
| 前端系统 | `http://localhost:18080` |
| 后端健康检查 | `http://localhost:4000/health` |
| EMQX 控制台 | `http://localhost:18083` |
| MySQL | `127.0.0.1:3307` |

EMQX 默认账号密码：

```text
admin / public
```

MySQL 默认连接信息：

```text
database: smart_streetlight
username: streetlight
password: streetlight_pass
port: 3307
```

系统默认登录账号：

```text
admin / admin123
```

正式交付前必须通过环境变量修改 `ADMIN_PASSWORD`、`JWT_SECRET`，并按项目需要配置 `AUTH_USERS_JSON`：

```env
AUTH_USERS_JSON=[{"username":"operator","password":"operator123","role":"operator"},{"username":"viewer","password":"viewer123","role":"viewer"}]
```

如需启用 MaxKB 智能问答，在 `.env` 或启动环境中配置：

```env
MAXKB_API_KEY=sk-...
MAXKB_BASE_URL=http://127.0.0.1:8080/api/application/your-application-id
MAXKB_MODEL=maxkb
MAXKB_TIMEOUT_MS=8000
```

未配置 `MAXKB_API_KEY` 时，问答接口会自动使用本地 RAG 检索兜底。

权限模型：

| 角色 | 权限 |
| --- | --- |
| `admin` | 全部读写权限，可新增设备和查看审计日志 |
| `operator` | 可调整阈值、下发控制、处理告警 |
| `viewer` | 只读查看总览、设备、告警和智能问答 |

## 6. 启动后必做验收

检查容器状态：

```bash
docker compose -f deploy/docker-compose.yml ps
```

检查后端健康状态：

```bash
curl -sS http://127.0.0.1:4000/health
```

期望包含：

```json
{
  "ok": true,
  "storageDriver": "mysql"
}
```

检查前端入口：

```bash
curl -sS -I http://127.0.0.1:18080
```

期望返回 `HTTP/1.1 200 OK`。

本地开发模式下检查前端入口：

```bash
curl -sS -I http://127.0.0.1:5173
```

期望返回 `HTTP/1.1 200 OK` 或 `HTTP/1.1 304 Not Modified`。

检查登录接口：

```bash
TOKEN=$(curl -sS -X POST http://127.0.0.1:4000/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"admin123"}' \
  | node -e "let data=''; process.stdin.on('data', c => data += c); process.stdin.on('end', () => console.log(JSON.parse(data).token));")
```

检查受保护 API：

```bash
curl -sS -H "Authorization: Bearer $TOKEN" http://127.0.0.1:4000/api/overview
```

检查审计接口：

```bash
curl -sS -H "Authorization: Bearer $TOKEN" http://127.0.0.1:4000/api/audit-logs
```

检查 viewer 越权拦截时，先通过 `AUTH_USERS_JSON` 配置 viewer 后重启后端，再获取 viewer token，并执行：

```bash
curl -sS -o /tmp/viewer-denied.json -w "%{http_code}" \
  -X POST http://127.0.0.1:4000/api/devices/SL-001/commands \
  -H "Authorization: Bearer $VIEWER_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"command":"TURN_ON"}'
```

期望返回 HTTP `403`，随后管理员查询 `/api/audit-logs` 可看到 `DENIED` 记录。

前端页面验收：

1. 打开 `http://localhost:18080`，本地开发则打开 `http://localhost:5173`。
2. 使用 `admin / admin123` 登录。
3. 在总览页确认设备数量、在线数、告警数和光照趋势能显示。
4. 在设备台账点击设备控制按钮，确认能打开详情并下发开灯/关灯。
5. 在区域管控页选择区域并下发命令，确认页面出现匹配设备和命令结果。
6. 在告警中心点击处理，填写备注后确认告警状态变化。
7. 在智能问答页提问，确认能返回 MaxKB 或本地 RAG 答案。

## 8. 常用运维命令

查看日志：

```bash
docker compose -f deploy/docker-compose.yml logs -f backend
docker compose -f deploy/docker-compose.yml logs -f frontend
```

重启服务：

```bash
docker compose -f deploy/docker-compose.yml restart backend frontend
```

停止服务但保留 MySQL 数据：

```bash
docker compose -f deploy/docker-compose.yml down
```

停止服务并清空 MySQL 数据卷：

```bash
docker compose -f deploy/docker-compose.yml down -v
```

## 9. 数据备份与恢复

备份当前 MySQL 数据库：

```bash
npm run db:backup
```

默认输出到：

```text
backups/mysql/smart-streetlight-YYYYMMDD-HHMMSS.sql
```

备份脚本通过 Compose 容器执行：

```bash
docker compose -f deploy/docker-compose.yml exec -T mysql sh -lc 'MYSQL_PWD="$MYSQL_PASSWORD" mysqldump --single-transaction --routines --triggers --events --no-tablespaces --default-character-set=utf8mb4 -u"$MYSQL_USER" "$MYSQL_DATABASE"'
```

恢复数据库：

```bash
npm run db:restore -- --input backups/mysql/smart-streetlight-YYYYMMDD-HHMMSS.sql --yes
```

恢复脚本会覆盖当前 MySQL 数据，未传入 `--yes` 时会直接拒绝执行。正式客户环境应将 `backups/mysql/` 同步到服务器外部存储或客户指定备份位置。

演练命令：

```bash
npm run db:backup -- --dry-run
npm run db:restore -- --input backups/mysql/example.sql --dry-run
```

## 10. 常见启动问题

| 现象 | 优先检查 |
| --- | --- |
| 前端页面能打开，但登录失败 | 后端 `4000` 是否启动，`curl http://127.0.0.1:4000/health` 是否正常 |
| 按钮点击后没有业务变化 | 对应后端接口是否启动、当前账号是否有权限、浏览器控制台是否有 401/403/500 |
| 实时数据不刷新 | mock-device 是否启动，EMQX `1883` 是否可连接，Socket.IO 是否连到后端 |
| 后端启动后数据库报错 | MySQL 容器是否 healthy，`.env` 中 `DATABASE_URL` 端口是否是 `3307` |
| AI 问答返回本地答案 | `MAXKB_API_KEY` 或 `MAXKB_BASE_URL` 未配置，或 MaxKB 接口不可达 |
| Docker 前端打开空白或旧页面 | 重新执行 `docker compose -f deploy/docker-compose.yml up -d --build frontend` |

如果遇到“页面上有按钮但点了没反应”，按这个顺序排查：

1. 浏览器控制台是否有报错。
2. Network 面板是否发出了 `/api/...` 请求。
3. 请求是否返回 `401`、`403`、`404` 或 `500`。
4. 后端是否存在对应接口。
5. 当前功能是否需要 MQTT、数据库或 MaxKB 等外部服务。
6. 如果按钮只是装饰或没有必要，应改成普通展示元素，不保留假按钮。

## 11. 生产注意事项

- 当前默认密码用于课程/实习交付，正式上线前必须更换系统管理员、JWT、MySQL 和 EMQX 密码。
- 当前已提供管理员、运维员、只读角色和审计日志；正式上线前需要结合客户组织结构配置真实账号。
- MySQL 数据保存在 Docker volume `deploy_mysql-data`，迁移服务器前必须先执行数据库备份，并在新环境完成恢复演练。
